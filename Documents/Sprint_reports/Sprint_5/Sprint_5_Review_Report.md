# Sprint 5 review report

## Sprint Goal

The objective of this sprint is to improve the ClassIQ application to provide support for 
multiple languages. This involves externalizing the user interface text, integrating the 
localization framework, supporting dynamic language switching, and supporting non-Latin 
scripts such as Arabic with RTL support and Sinhalese. The objective is also to ensure that 
the application is scalable to add more languages in the future.


## Completed User Stories / Tasks

- Sprint 5 planning.
- Identified and extracted all hardcoded UI text into separate translation files.
- Refactored the codebase to load UI text dynamically from the translation files.
- Developed structured translation files.
- Integrated the localization framework into the application.
- Developed the feature to enable users to switch languages using the language selector.
- Developed the feature to enable dynamic language switching without requiring page reload.
- Added Arabic language support with proper RTL layout adjustments.
- Added Sinhalese language support, the second non-Latin language.
- Made UI component adjustments to properly handle RTL alignment and layout.
- Upgraded the README file to include localization setup and usage instructions.
- Upgraded the project backlog with localization acceptance criteria.


## Demo Summary

During the sprint 5 review, the team demonstrated some of the completed features of the system. These features included the multi-lingual capabilities of the application, which included the language selector, language switching, and the successful integration of the localization framework. The team also demonstrated how the application supported multiple languages, including non-Latin scripts such as Arabic and Sinhalese.

## What Went Well

The team demonstrated good teamwork and collaboration, especially when working on complex feature implementations such as localization and RTL. All the tasks scheduled for Sprint 5 were accomplished without any delays. The localization framework and dynamic language switch integration worked well, and the application was able to run with multiple languages, including non-Latin scripts.

## What Could Be Improved

Time management could have been improved, as balancing academic responsibilities and sprint tasks was challenging. Moreover, more time could have been allocated to further enhance UI alignment and ensure consistency across languages, especially for RTL layouts.

## Next Sprint Focus

The next sprint will involve the implementation of database localization to extend the support of multiple languages beyond the current interface. The team will focus on modifying the database to support multiple languages, enable the retrieval of data based on user preferences, and improve the overall scalability of the system. Moreover, the team will also focus on optimizing the performance and ensuring consistency of the data in all supported languages.
