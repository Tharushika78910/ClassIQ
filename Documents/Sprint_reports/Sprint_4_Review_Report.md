# Sprint 4 review report

## Sprint Goal

The goal of this sprint 4 is to complete the prototype by finalizing its functionality, containerizing the application with Docker, building and publishing the Docker image to Docker Hub, testing the deployed image in an appropriate environment, sharing the project with classmates to gather feedback, and preparing for the final demonstration.


## Completed User Stories / Tasks

- Created the spring 4 planning.
- Completed all planned features and fixed remaining bugs in the system.
- Tested business logic accuracy.
- Build the Docker image using docker build and tested it locally using docker run to confirm everything works inside the container
- Pushed the Docker image to Docker Hub and verified its public availability.
- Tested the deployed image in Docker Play and confirmed it works as expected.
- Prepared a PowerPoint presentation to present the project to the class and collected feedback from classmates to improve the project.
- Updated the project documentation on GitHub and README file to reflect the final state of the project and submitted the final Sprint Report.


## Demo Summary

In the final presentation, all five team members took part in presenting the project. Roshini began by giving a brief introduction to the project objectives. She also explained the development tools and resources used, including how AI supported our project, and the Trello and GitHub links were included in the slides. Chathurika briefly presented the project backlog. Next, the Scrum Masters of each sprint explained the sprint backlogs, completed tasks, and testing activities carried out during the sprints. Dilmi then presented and explained the diagrams used in the project as well as the technologies applied. Poornima conducted the project demonstration, while Kumudu concluded by explaining the learning achievements and future plans.

## What Went Well

What went well was the strong teamwork shown during the final presentation. All five members contributed and explained their parts clearly.All Sprint 4 backlog tasks were successfully completed, and no tasks were postponed.

## What Could Be Improved

Time management was one area that could have been improved during the sprint. Because we had examinations at the same time, it was challenging to manage our time well. If we had more time, we could have improved the code quality even further.

## Next Sprint Focus

The next sprint will focus on improving the overall quality of the project.We plan to further enhance the user interface and add a new portal for parents. In addition, we hope to introduce new features to the system according to the requirements of the next stage of the project.
