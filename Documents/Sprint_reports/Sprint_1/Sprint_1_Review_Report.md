# Sprint 1 review report

## Sprint Goal

The goal of this sprint is to establish a project team, select a project topic, define the product vision and project plan, and set up project management tools along with the initial product backlog.

## Completed User Stories / Tasks

- Created a project team and selected a project topic and name.
- Set up a GitHub repository for code management and collaboration.
- Created a Trello workspace for project planning and tracking.
- Defined the product vision, including problem statement, target audience, value proposition, goals, and vision statement.
- Discussed and finalized user stories from the perspectives of teachers and students.
- Developed a UI blueprint for the application in Figma.
- Defined the project plan and timelines.

## Demo Summary

During the sprint review meeting, we presented the project setup, including the GitHub repository and Trello board. We showcased the defined project vision and project plan. Additionally, we demonstrated the UI blueprint created in Figma, highlighting the key features and user flow of the application.

## What Went Well

- Effective team collaboration in forming the project team and selecting the topic.
- Successful setup of project management tools (GitHub and Trello).
- Clear definition of project vision and user stories.

## What Could Be Improved

- Allocate more time for UI design 
- Plan for potential risks and challenges that may arise during the project.

## Next Sprint Focus

In the next sprint, we will focus on installing the development environment, and write unit tests, generate code coverage reports and apply clean code practices.
