# Sprint 1 review report

## Sprint Goal

The goal of this sprint 2 is to initiate the technical foundation of ClassIQ project by implementing the database using MariaDB,
starting the user interface
development with JavaFX , starting Backend programmes  and integrating development tools such as unit testing and code coverage reporting.


## Completed User Stories / Tasks

- Created the spring 2 planning.
- Set up database schema in MariaDB and configured JDBC connection in IntelliJ.
- Tested CRUD operations for student details in the database.
- Implemented the backend code for calculating total marks and average marks for students.
- Implemented the logic for determining the weightage of each subject and calculating the final grade based on the total marks.
- Initially set up the JavaFX project structure in IntelliJ and created the Home Page and Login Page.
- Implemented the Teacher's Dashboard with basic layout.
- Configured Maven dependencies for unit testing and wrote unit tests for the core application logic.
- Integrated JaCoCo for code coverage testing and generated a code coverage report.
- Committed all code changes to the GitHub repository and updated the Trello board with completed tasks and user stories.

## Demo Summary

In the demo review, we demonstrated our project’s readiness by presenting the database structure and stored data, showing the progress made on the user interface, and running unit tests to verify system functionality. We also explained how Maven is used for dependency management and project build, and shared the publicly available JaCoCo report to highlight our code coverage. Each team member clearly described their individual contributions to the project. In addition, we ensured that all recent changes were committed and updated on GitHub, and that our project progress and task status were properly reflected on Trello

## What Went Well

The project development progressed smoothly. The database was designed and implemented successfully. Unit tests were completed and executed properly, ensuring reliable system functionality. Maven dependency management and the build process worked efficiently without major issues. The JaCoCo coverage report was generated successfully, reflecting good test coverage. Good coordination among team members and updated gitHub and Trello helped in tracking progress and maintaining project organization.

## What Could Be Improved

Although the project progressed well, there are still some areas that can be improved. More time can be spent on enhancing the user interface to make it and  test coverage can be increased by adding more unit tests to improve system.

## Next Sprint Focus

In the next sprint, we will focus on implementing advanced features such as user authentication, data validation, search, and reporting. We tested and optimized core functionalities, fixed bugs, and improved system performance. Jenkins was integrated to automate building, testing, and code coverage reporting. We also expanded unit tests, prepared the system for functional review, updated documentation, and created and tested a Docker image for local deployment.
