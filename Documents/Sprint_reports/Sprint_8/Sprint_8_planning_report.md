## Planned Tasks/Breakdown

---

- Finalize GitHub Documentation

In Sprint 8, the team reviews and completes the full project documentation on GitHub. The README file is updated according to the instructions provided during the lectures. The team checks that the README clearly explains the project vision, purpose, features, installation steps, usage instructions, technologies used, and team contributions.

- Update Docs Folder

The Docs folder is modified and expanded to include all important project documents from the 16-week development period. This includes sprint review reports, UML diagrams, ER diagrams, Clean Code documentation, Static Code Review results, Heuristic Evaluation results, and UAT test results. The team also checks that all files are clearly named, correctly organized, and easy to understand.

- Prepare Final Presentation 

The team prepares a 25-minute PowerPoint presentation using the provided template. The presentation summarizes the project vision, problem, solution, main features, implementation process, testing results, and final outcome.


## Team Capacity & Assumptions

---

The team members have completed the main implementation work during the previous sprints and are now focusing on documentation, presentation preparation, and final submissions. It is assumed that all required project materials are available in GitHub or can be added to the Docs folder

## Definition of Done


___


A backlog item is considered done when the README is fully updated, the Docs folder contains all required project documents, and all diagrams, review reports, code review documents, heuristic evaluation results, and UAT results are included. The final presentation must be completed using the provided template and prepared for a delivery with active participation from all team members.
