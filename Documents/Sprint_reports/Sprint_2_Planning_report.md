# Sprint 2 ( 28/01/2025 - 10/02/2025 )

---

## Sprint Goal

---

The goal of this sprint 2 is to initiate the technical foundation of ClassIQ project by implementing the database using MariaDB,
starting the user interface 
development with JavaFX , starting Backend programmes  and integrating development tools such as unit testing and code coverage reporting.


## Selected Product Backlog Items

---

- Design relational database schema
- Start user interface 
- Begin Backend development
- Database connectivity
- Unit test framework
- Code coverage reporting


## Planned Tasks/Breakdown

---

 - Design relational database schema  
  
    This task includes designing database schema by defining entities , attributes and relationship,
    creating the required tables in MariaDB,and configuring JDBC connection in intelliJ to enable communication between the application and the database.
 
   
 - Start user interface  
      This task involves creating javaFX project structure in IntelliJ,
      implementing the initial screen such as Home Page, Login Page ,Teacher's Dash board ,
      and applying layouts according to the Figma design.

   
 - Begin Backend development   
   We start the Backend development by writing the core code in IntelliJ.First ,we 
   create functions to enter and manage student details such as student name and student ID ,
    which will be used later to store and process student information in the system.



 - Integrate Unit Test  
   This task involves configure Maven Dependencies for unit, writing unit test for core application logic,
   and committing test files to the GitHub repository.


 - Configure Code Coverage Testing  
    This task involves integrating JaCoCo into the project,
   running the tests and generating a code coverage report to track which parts of the code are tested.


## Team Capacity & Assumptions

---

The team members are available throughout the sprints and have basic knowledge of Java ,IntelliJ and SQL.

In class activities and meetings support sprint tasks,which include creating the ER model,writing JUnit tests for core logic
, and generating code coverage report.


## Definition of Done


___


A task is considered complete when the initial code works without errors, the database and UI are functional, tests pass 
with a generated JaCoCo report ,and all updates are committed to GitHub and tracked in Trello.
